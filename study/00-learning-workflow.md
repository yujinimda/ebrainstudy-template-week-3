# 학습 루틴 (이슈 하나당)

> 이 리포는 "동작하는 코드"보다 "내가 이해하고 직접 짠 코드"가 목표다.

## 진행 순서

1. **개념 설명** — Claude가 이번 이슈에 필요한 개념을 짧게, 지금 짤 코드와 연결해서 설명한다.
2. **퀴즈** — 설명한 개념으로 적용형 문제 1~3개. 통과해야 다음 단계로.
3. **구현** — 역할 분담:
   - 핵심 로직(REST 설계, 쿼리, 검증, 상태 관리, 데이터 흐름)은 **내가** 짠다. Claude는 `TODO(human)` 주석으로 자리만 남긴다.
   - 보일러플레이트(설정 파일, 반복 마크업, DTO getter류)는 Claude가 짠다.
4. **리뷰** — 내가 짠 코드를 Claude가 리뷰. 틀리면 정답 대신 "어디가 왜 문제인지"만 짚는다. 두 번 이상 막히면 그때 답을 본다.
5. **노트** — 막혔던 것/중요 개념을 `study/notes/issue-NN-<slug>.md`로 남긴다. (`study/_template.md` 복사)
6. **PR** — 이슈당 브랜치 하나 → PR → 머지. PR 본문에 노트 링크.

## 모노레포에서 일하기

- 백엔드 작업은 `backend/`, 프론트 작업은 `frontend/` 안에서 한다.
- 한 이슈가 백/프론트를 모두 건드리면(예: 등록 폼) PR 하나에 양쪽 변경을 함께 담는다.
- 실행: 백엔드 `cd backend && ./gradlew bootRun`, 프론트 `cd frontend && npm run dev`, DB `cd docker && docker compose up -d`.

## 브랜치/커밋 규칙 (2주차와 동일)

- 브랜치: `feature/issue-NN-<slug>`
- 커밋 메시지 첫 줄에 이슈 번호: `#NN 목록 API 작성`

## 폰에서 복습하기

- 이슈 본문 + `study/notes/`가 전부 GitHub에 있으므로 **폰 GitHub 앱**에서 바로 읽을 수 있다.
- **폰 Claude 앱**: claude.ai에서 GitHub 연동으로 이 리포를 연결하면, 노트를 근거로 질문/복습 대화를 이어갈 수 있다.
